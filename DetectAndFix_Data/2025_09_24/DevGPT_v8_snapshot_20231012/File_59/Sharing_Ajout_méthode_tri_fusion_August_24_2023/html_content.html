error: use of deleted function 'std::unique_ptr<_Tp, _Dp>::unique_ptr(const std::unique_ptr<_Tp, _Dp>&) [with _Tp = Node<int>; _Dp = std::default_delete<Node<int> >]' fast = fast->next ? fast->next->next : nullptr;

L'erreur que vous rencontrez est due à la tentative de copier un std::unique_ptr, ce qui n'est pas autorisé. Les std::unique_ptr ne peuvent pas être copiés car ils garantissent une propriété unique de l'objet pointé. Vous pouvez seulement transférer la propriété d'un std::unique_ptr à un autre en utilisant std::move.

L'erreur est déclenchée par cette ligne :

cpp
fast = fast->next ? fast->next->next : nullptr;

Ici, fast->next et fast->next->next sont des std::unique_ptr, donc vous ne pouvez pas directement les affecter à fast.

La solution est d'utiliser des pointeurs bruts (Node<T>*) pour les pointeurs temporaires fast et slow lors du fractionnement. Ces pointeurs bruts ne possèdent pas la mémoire, ils la pointent simplement, donc cela ne pose pas de problème.

Voici comment vous pouvez corriger la méthode frontBackSplit :

cpp
void frontBackSplit(std::unique_ptr<Node<T>>& source, std::unique_ptr<Node<T>>& frontRef, std::unique_ptr<Node<T>>& backRef)
{
    Node<T>* fast = source.get();
    Node<T>* slow = source.get();

    while (fast && fast->next)
    {
        slow = slow->next.get();
        fast = fast->next->next ? fast->next->next.get() : nullptr;
    }

    frontRef = std::move(source);
    backRef = std::move(slow->next);
}

Avec cette modification, la méthode frontBackSplit devrait fonctionner correctement sans tenter de copier un std::unique_ptr.
