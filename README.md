# Secure Coding with AI – From Detection to Repair

## Vulnerability Analysis and Fixing Using Large Language Models

Publicly available data for the *"Secure Coding with AI – From Detection to Repair"* paper

The software tools created and used in this study are available in the repository:
https://github.com/vlad2010/MS_DataEngineering_Public

## Content description

### OpenAI_API_Issues64_Experiment subfolder

* OpenAI_API_Issues64_Experiment\2024_10_22 - contains results of the first experiment (October 2024)
* OpenAI_API_Issues64_Experiment\SecurityIssueAnalysis_2024.ods - LibreCalc table with detailed manual analysis results.

#### Each File_XX subfolder in OpenAI_API_Issues64_Experiment\2024_10_22 contains:

* DevGPTData - folder with information from the DevGPT dataset
* Info.txt - static scanner detection information
* New_generated_code.c - code generated by GPT-4o; file extension can be c, cpp, or cs
* OurPrompt.txt - prompt used
* ChatGPT_Responce.md - answer to the prompt generated by the LLM in Markdown format
* Code_XXX.c - original source file from DevGPT

### DetectAndFix_Data subfolder 

DetectAndFix_Data\2025_09_24\ - contains results of the second experiment (September 2025)

* Readme.md - information about the date of the run and the LLM used
* SecurityIssuesAnalysis.ods - LibreCalc table with detailed manual analysis results
* llm_config.json - configuration file

#### DevGPT_v8_snapshot_20231012 and DevGPT_v10_snapshot_20240514 subfolders

##### General content

* info.txt - static scanner detection information
* source_data_info.txt - metadata from DevGPT 
* Sharing_<name> - folder containing DevGPT data; full series of Conversation folders for
* LLMs - subfolder with data for specific LLM

  * claude-opus-4-1-20250805 - answers generated by Claude Opus 4.1
  * gpt-4.1-2025-04-14 - answers generated by GPT-4.1
  * gpt-5-2025-08-07 - answers generated by GPT-5

##### LLM-specific content

Each LLM folder contains the following data:

* Code_XXX.c - original source file from DevGPT; extension can be c, cpp, or cs
* LLM_Responce.md - answer to the prompt generated by the LLM in Markdown format
* New_generated_code_XX.c - new code generated by the LLM; file extension can be c, cpp, or cs
* OurPrompt.txt - prompt used
