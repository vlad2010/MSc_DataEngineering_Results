UDP Byte Order Issue

July 20, 2023
udp 프로토콜을 사용하고 내가 server야. client가 16진수로 aa 03 da 00 01을 보내주는데 server는 da03aa로 받고있어. 뭐가 문제일까

UDP 프로토콜을 사용하는 서버와 클라이언트 간에 데이터 통신 시에는 데이터의 바이트 순서(Byte Order)에 주의해야 합니다. 데이터를 올바르게 해석하기 위해서는 클라이언트와 서버 간에 바이트 순서를 동일하게 맞추어야 합니다.

보고하신 상황에서 문제는 클라이언트가 보내는 데이터와 서버가 받는 데이터의 바이트 순서가 일치하지 않기 때문입니다. 클라이언트가 보내는 데이터는 16진수 aa 03 da 00 01 이지만, 서버가 받는 데이터는 da03aa 입니다.

이러한 문제를 해결하기 위해서는 데이터를 보낼 때 클라이언트와 서버가 동일한 바이트 순서를 사용하도록 합니다. 보통 네트워크 통신에서 사용하는 바이트 순서는 두 가지가 있습니다.

  1. 빅 엔디안(Big Endian): 가장 높은 자릿수의 바이트를 먼저 보내는 방식
  2. 리틀 엔디안(Little Endian): 가장 낮은 자릿수의 바이트를 먼저 보내는 방식

예를 들어, 클라이언트가 빅 엔디안 방식으로 데이터를 보내고 있지만, 서버는 리틀 엔디안 방식으로 데이터를 받고 있는 것 같습니다. 이를 해결하기 위해서는 클라이언트와 서버가 동일한 바이트 순서를 사용하도록 설정해야 합니다.

일반적으로 네트워크 통신에서는 빅 엔디안을 더 많이 사용합니다. 따라서 클라이언트나 서버 중 한 쪽을 빅 엔디안으로 설정하면 될 것입니다. 물론 어느 쪽을 선택하더라도 중요한 것은 두 쪽이 동일한 바이트 순서를 사용하도록 맞추는 것입니다.

서버를 빅 엔디안으로 어떻게 바꾸지? 라즈비안 os에서 c언어로 프로그래밍했어

데이터의 바이트 순서를 빅 엔디안으로 바꾸는 방법은 C 언어에서도 간단하게 수행할 수 있습니다. C 언어에서는 특정 데이터를 빅 엔디안으로 변환하기 위해 htonl (Host to Network Long)과 같은 함수를 사용합니다. 또한, 받은 데이터를 다시 호스트 바이트 순서로 변환하기 위해 ntohl (Network to Host Long) 함수를 사용할 수 있습니다.

빅 엔디안으로 변환한 데이터를 클라이언트에서 보내면 서버는 빅 엔디안으로 올바르게 받을 수 있을 것입니다. 마찬가지로 서버에서 클라이언트로 보낼 때도 htonl과 htons 함수를 사용하여 데이터를 변환하여 보내주어야 합니다.

이와 유사한 함수들은 Windows 환경에서도 사용할 수 있으며, Windows에서는 htonl 대신 htonl과 같은 WinSock 함수를 사용합니다. 하지만 라즈비안 OS에서는 주로 POSIX 표준의 함수를 사용하므로 위의 예제 코드를 사용해도 잘 작동할 것입니다.

#include <stdio.h> #include <stdlib.h> #include <string.h> #include <unistd.h> #include <arpa/inet.h> #define PORT 3001 #define BUFFER_SIZE sizeof(int) int main() { int server_socket; struct sockaddr_in server_addr, client_addr; socklen_t client_addr_size = sizeof(client_addr); long int number; int flag, c; // Create socket if ((server_socket = socket(AF_INET, SOCK_DGRAM, 0)) < 0) { perror("socket creation failed"); exit(EXIT_FAILURE); } memset(&server_addr, 0, sizeof(server_addr)); memset(&client_addr, 0, sizeof(client_addr)); // Configure server address server_addr.sin_family = AF_INET; server_addr.sin_port = htons(PORT); server_addr.sin_addr.s_addr = INADDR_ANY; // Bind the socket to the server address if (bind(server_socket, (const struct sockaddr *)&server_addr, sizeof(server_addr)) < 0) { perror("bind failed"); exit(EXIT_FAILURE); } printf("Server started. Waiting for messages...\n"); while (1) { // Receive number from client recvfrom(server_socket, &number, sizeof(long int), 0, (struct sockaddr *)&client_addr, &client_addr_size); printf("Client: %lx\n", number); // Reply to client do{ flag=0; printf("Server (You): "); if(!scanf("%lx", &number)){ printf("숫자를 입력하세요."); flag=1; while(c=getchar() != '\n' && c!= EOF); } }while(flag); sendto(server_socket, &number, sizeof(int), 0, (const struct sockaddr *)&client_addr, client_addr_size); } close(server_socket); return 0; }

주어진 코드는 UDP를 사용하여 클라이언트와 통신하는 간단한 서버 코드입니다. 이 코드는 클라이언트로부터 데이터를 수신하고 다시 클라이언트에게 데이터를 보내는 간단한 에코 서버의 역할을 합니다. 또한, 숫자를 받고 보낼 때 데이터의 바이트 순서를 고려하지 않으므로, 서버와 클라이언트가 동일한 바이트 순서를 사용하고 있다고 가정합니다.

다만, 주의할 점이 있습니다. recvfrom 함수와 sendto 함수에서 사용되는 데이터 크기가 다릅니다. recvfrom 함수에서 데이터를 받을 때 sizeof(long int) 크기의 데이터를 받지만, sendto 함수에서는 sizeof(int) 크기의 데이터를 보냅니다. 이로 인해 데이터의 크기가 다른 문제가 발생할 수 있습니다.

이러한 이유로, recvfrom과 sendto 함수에서 사용하는 데이터의 크기를 일치시켜 주시기 바랍니다. 예를 들어 recvfrom 함수에서 받을 때도 sizeof(int) 크기의 데이터를 받도록 변경하시면 됩니다.

추가적으로, recvfrom 함수의 호출은 데이터를 수신하고 있지만, 해당 코드에서 클라이언트로부터 데이터를 받기 전에 서버에서 사용자로부터 숫자를 입력받는 부분이 있습니다. 이 부분은 현재 서버가 클라이언트로부터 데이터를 받기 전까지 무한히 대기하게 되어 있습니다. 이를 해결하려면 클라이언트로부터 데이터를 받는 부분과 사용자로부터 숫자를 입력받는 부분을 별도의 쓰레드로 처리하거나, 비동기적인 방식으로 구현해야 합니다.

마지막으로, 데이터를 빅 엔디안으로 변환하는 부분을 추가해야 한다고 언급한 것처럼, 데이터의 바이트 순서에 주의하시기 바랍니다. recvfrom 함수로 수신한 데이터가 클라이언트에서 보낸 데이터와 일치하도록 바이트 순서를 맞춰주어야 합니다.

수정된 코드를 살펴보면, recvfrom 함수와 sendto 함수에서 사용되는 데이터의 크기가 sizeof(int)으로 일치하도록 변경되었습니다. 이제 데이터의 크기 문제는 해결되었습니다.

또한, %lx를 %x로 변경하여 printf 함수에서 16진수로 출력하도록 수정되었습니다. 이제 서버는 클라이언트로부터 정상적으로 16진수 데이터를 받아서 출력하고, 사용자로부터도 16진수 숫자를 입력받아서 클라이언트에게 다시 보내는 기능을 수행할 수 있을 것입니다.

다만, 여전히 데이터의 바이트 순서에 대한 처리는 이루어지지 않았습니다. 기본적으로 C 언어의 데이터 타입은 호스트 바이트 순서로 처리되기 때문에, 이 코드가 작동하기 위해서는 클라이언트와 서버 모두 호스트 바이트 순서로 데이터를 처리해야 합니다.

%02x가 뭐야

%02x는 C 언어의 printf 함수에서 사용되는 서식 지정자입니다. 이 서식 지정자는 정수 값을 2자리 16진수로 표현하는 데 사용됩니다. 각각의 부분에 대해 설명드리겠습니다.

  • %: 서식 지정자의 시작을 나타냅니다.
  • 0: 빈 자리를 0으로 채우도록 지정합니다.
  • 2: 최소 필드 너비를 2로 지정합니다. 만약 16진수 값이 1자리이면 앞에 0을 붙여 2자리로 만듭니다.
  • x: 16진수로 출력하도록 지정합니다.

서식 지정자 %02x를 예시로 설명하면, %02x는 2자리의 16진수를 표현하는 데 사용됩니다. 예를 들어, 16진수 값이 7인 경우 07로 출력하고, 16진수 값이 12인 경우 0c로 출력합니다. 이렇게 %02x를 사용하면 16진수 값을 항상 2자리로 표현할 수 있습니다.
